Static Site Boilerplate

Introduction

A repository I can git-clone when starting new projects.

Pretty similar to Yeoman, but pared down to personal preferences.

Powered heavily by grunt, which has revolutionized my front-end workflow.

Supported Features

  • Modernizr.js for feature detection
  • jQuery (local on dev, Google CDN on production)
  • Coffeescript compilation
  • SASS/Compass compilation
  • Production: Minification of CSS/JS, update references in HTML
  • Production: Image optimization

Commands

  • 'grunt dev': Spin up a webserver, watch the assets/coffeescript and assets/sass directories for changes and compile accordingly. Start a local webserver at localhost:9001.
  • 'grunt prod': Minify js files, optimize images. Copy all necessary .html, minified css/js, and optimized assets to _site directory. Update the references to those assets in the html file. Start a local webserver at localhost:9001.
